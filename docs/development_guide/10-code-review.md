## 一、审查原则

### 1.1 审查目标

- **质量**: 发现缺陷、提升代码质量
- **知识**: 团队知识共享、技术传播
- **一致**: 保持代码风格一致
- **成长**: 帮助团队成员成长

### 1.2 审查态度

- **尊重**: 对事不对人，尊重作者劳动
- **建设性**: 提出改进建议，而非仅批评
- **及时性**: 24小时内完成审查
- **完整性**: 全面审查，不走过场

## 二、审查清单

### 2.1 功能性审查

- [ ] 代码是否实现了需求
- [ ] 边界条件是否处理
- [ ] 错误处理是否完善
- [ ] 并发安全是否考虑
- [ ] 性能是否可接受

### 2.2 可读性审查

- [ ] 命名是否清晰有意义
- [ ] 函数是否简短专注
- [ ] 注释是否必要且准确
- [ ] 代码结构是否清晰
- [ ] 复杂度是否可控

### 2.3 可维护性审查

- [ ] 是否遵循设计原则 (SOLID)
- [ ] 是否有重复代码
- [ ] 是否可测试
- [ ] 是否有技术债务
- [ ] 文档是否同步更新

### 2.4 安全性审查

- [ ] 输入是否验证
- [ ] 敏感数据是否保护
- [ ] 是否有注入风险
- [ ] 权限控制是否正确
- [ ] 日志是否脱敏

## 四、评论规范

### 4.1 评论等级

| 等级 | 图标 | 说明 | 必须修复 |
|------|------|------|----------|
| **阻断** | 🔴 | 严重问题，必须修复 | 是 |
| **建议** | 🟡 | 改进建议，推荐修复 | 否 |
| **疑问** | 🔵 | 需要澄清的问题 | 视情况 |
| **赞赏** | 🟢 | 好的实践，给予肯定 | 否 |

### 4.2 评论示例

#### 阻断级评论

```
🔴 **安全问题**: 这里直接使用用户输入拼接 SQL，存在 SQL 注入风险。

建议:
```python
# 错误
cursor.execute(f"SELECT * FROM users WHERE id = {user_id}")

# 正确
cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))
```
```

#### 建议级评论

```
🟡 **可读性**: 这个函数较长，建议拆分成更小的函数。

当前函数处理了:
1. 数据验证
2. 数据转换
3. 数据库操作
4. 缓存更新

建议拆分为:
- validate_data()
- transform_data()
- save_to_db()
- update_cache()
```

#### 疑问级评论

```
🔵 **疑问**: 这里为什么要用递归而不是循环?

递归可能导致栈溢出，如果是性能考虑，能否说明一下?
```

#### 赞赏级评论

```
🟢 **赞赏**: 这个错误处理很完善，包含了用户友好的错误信息!
```

## 五、审查工具

### 5.1 自动化检查

| 工具 | 用途 | 集成方式 |
|------|------|----------|
| **pre-commit** | 提交前检查 | Git hook |
| **GitHub Actions** | CI检查 | PR触发 |
| **SonarQube** | 代码质量 | 定时扫描 |
| **CodeClimate** | 技术债务 | PR检查 |

### 5.2 CI检查配置

```yaml
# .github/workflows/pr-check.yml
name: PR Check

on:
  pull_request:
    branches: [main, develop]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      
      - name: Run linter
        run: |
          pip install pylint black
          pylint app/
          black --check app/
      
      - name: Run tests
        run: |
          pip install pytest pytest-cov
          pytest --cov=app --cov-report=xml
      
      - name: Check coverage
        run: |
          coverage=$(cat coverage.xml | grep -o 'line-rate="[0-9.]*"' | head -1 | cut -d'"' -f2)
          if (( $(echo "$coverage < 0.80" | bc -l) )); then
            echo "Coverage $coverage is below 80%"
            exit 1
          fi
```
