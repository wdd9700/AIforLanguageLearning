# 04 - 边界澄清

> **开发方法论**: Agentic Engineering + BMAD-METHOD + SDD  
> **版本**: v1.0  
> **最后更新**: 2026-03-30

---

## 一、系统边界

### 1.1 功能边界

#### 包含功能 (In Scope)

| 模块 | 功能范围 |
|------|----------|
| **单词** | 查词、词库生成、标签归类、推荐复习 |
| **作文** | OCR提取、GEC纠错、LLM评分、反馈展示 |
| **对话** | 场景扩写、语音识别、AI回复、语音合成 |
| **教师端** | 学习数据分析、进度追踪、班级管理 |
| **实时助教** | 屏幕捕获、行为识别、智能建议、语音输出 |

#### 不包含功能 (Out of Scope)

| 功能 | 说明 | 替代方案 |
|------|------|----------|
| 视频课程 | 录播视频播放 | 集成第三方 |
| 直播课堂 | 实时视频教学 | 集成腾讯会议/钉钉 |
| 支付系统 | 课程购买支付 | 集成微信支付/支付宝 |
| 内容审核 | UGC内容审核 | 集成云审核服务 |
| 多语言支持 | 除英语外的语言 | V2版本规划 |

### 1.2 技术边界

| 层级 | 边界定义 |
|------|----------|
| **前端** | Vue 3 + TypeScript，不支持IE11 |
| **后端** | Python 3.10+，FastAPI框架 |
| **AI推理** | 本地GPU(5080) + 云端Kimi API |
| **数据库** | PostgreSQL 15+，Redis 7+ |
| **部署** | Docker容器化，Linux主机 |

### 2.4 实时助教模块边界 **[最后开发，最亮点，最难]**

```
┌─────────────────────────────────────────────────────────────┐
│                    实时助教模块边界                          │
├─────────────────────────────────────────────────────────────┤
│  【核心设计原则】                                             │
│  • 屏幕处理: 720P下采样，保留鼠标轨迹(非ROI裁剪)              │
│  • 筛选机制: 五级筛选(L0-L4)，节省97% Token                  │
│  • 唤醒机制: ASR+关键词(显式+隐式信号)                        │
│  • 服务形式: 被动+主动(轻量提示/主动辅助/重要提醒)            │
│  • 投递策略: 无感投递(停顿/翻页/句子结束检测)                 │
│                                                              │
│  【前端 - 教师端PC】                                          │
│  L0: 输入事件捕获                                            │
│  • 键盘: PageUp/PageDown/方向键 → 翻页检测                    │
│  • 鼠标: 按下+移动轨迹 → 框选行为检测                          │
│  • 滚轮: 滚动方向/速度 → 浏览行为检测                         │
│                                                              │
│  L1: 超轻量行为识别 (WebGL/ONNX Runtime)                      │
│  • 模型: YOLOv8n 6MB 或 MobileNetV3-SSD 3MB                  │
│  • 输入: 鼠标轨迹 + 屏幕区域                                  │
│  • 输出: 框选区域坐标 [x1,y1,x2,y2] + 置信度                  │
│  • 延迟: < 10ms                                              │
│                                                              │
│  L2: 屏幕内容哈希去重                                         │
│  • 感知哈希 (pHash/dHash) 快速对比                            │
│  • 分辨率下采样: 1080P→720P (保留鼠标轨迹)                    │
│  • 相似帧过滤 (动画、过渡效果)                                │
│                                                              │
│  【后端 - 本地服务】                                          │
│  L3: 注意力Mask生成与语义分析                                 │
│  • 框选区域 → 半透明高亮Mask (引导VLM注意力)                   │
│  • OCR提取框选区域内文本                                      │
│  • 翻页事件 → 课件段落定位                                    │
│                                                              │
│  L4: 智能触发决策引擎                                         │
│  • 高优先级: 框选行为 + 语音停顿 → 立即触发VLM                 │
│  • 中优先级: 翻页 → 延迟触发                                  │
│  • 低优先级: 仅翻页/仅语音 → 本地规则兜底                      │
│  • 忽略: 快速翻页、随机晃动、UI操作                            │
│                                                              │
│  【AI服务】                                                   │
│  • VLM: Kimi K2.5 (统一多模态模型)                           │
│  • 输入: 720P截图 + 注意力Mask + ASR文本 + 结构化上下文        │
│  • 输出: 结构化助教建议                                       │
│                                                              │
│  【输出】                                                     │
│  • 语音: XTTS本地合成 → WebSocket → 教师耳机                  │
│  • 视觉: 提示图标/文字 → WebSocket → 教师屏幕                 │
│                                                              │
│  【重要约束】                                                 │
│  • 仅教师端使用，学生端完全无感知                             │
│  • 所有输出仅返回给教师，不干扰正常教学                       │
│  • 端到端延迟目标: < 800ms                                   │
│  • Token成本控制: 45分钟课程~15次请求                         │
└─────────────────────────────────────────────────────────────┘
```

## 四、接口边界

### 4.1 内部接口

| 接口 | 协议 | 认证 | 限流 |
|------|------|------|------|
| 前端↔后端 | HTTP/WebSocket | JWT | 1000/min |
| 后端↔AI | HTTP/gRPC | API Key | 按模型 |
| 后端↔数据库 | TCP | 密码 | 连接池 |

### 4.2 外部接口

| 接口 | 用途 | 稳定性要求 |
|------|------|------------|
| Kimi API | LLM/VLM推理 | 必须有降级 |
| 微信支付 | 支付功能 | 可选功能 |
| 短信服务 | 验证码 | 可切换供应商 |
