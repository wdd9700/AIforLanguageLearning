## 四、性能优化策略

### 4.1 缓存策略

| 层级 | 内容 | TTL | 命中率目标 |
|------|------|-----|------------|
| L1 | 进程内配置 | 无 | - |
| L2 | Redis查词 | 1小时 | > 80% |
| L2 | Redis会话 | 30分钟 | - |
| L3 | CDN静态资源 | 24小时 | > 90% |

### 4.2 实时助教优化策略

| 优化点 | 策略 | 效果 |
|--------|------|------|
| **屏幕传输** | 720P下采样，保留鼠标轨迹 | 节省75%带宽 |
| **行为识别** | 前端WebGL/ONNX，6MB模型 | <10ms延迟 |
| **触发筛选** | 五级筛选(L0-L4) | 节省97% Token |
| **注意力引导** | 半透明Mask叠加 | 提升VLM准确率 |
| **ASR唤醒** | 关键词检测+语义分析 | 精准触发 |

### 4.3 批处理策略

| 场景 | 批大小 | 超时 |
|------|--------|------|
| 模型推理 | 8-16 | 100ms |
| 数据库写入 | 100 | 1s |
| 日志收集 | 1000 | 5s |

### 4.3 降级策略

| 场景 | 降级方案 | 触发条件 |
|------|----------|----------|
| Kimi API超时 | 本地9B模型 | > 2s |
| GPU满载 | CPU推理 | 显存>95% |
| 数据库慢 | 缓存返回 | > 500ms |
| 服务过载 | 限流+排队 | QPS>阈值 |
