# AI外语学习系统 - 模块详细说明与信息流向

## 一、核心服务模块（6个）

### 1. 数据库服务 (PostgreSQL + TimescaleDB)

**功能职责**：
- 存储用户账号信息、学习记录、单词数据
- 存储作文内容、批改结果、对话历史
- 存储用户行为数据（用于画像分析）
- TimescaleDB扩展处理时间序列数据（学习进度追踪）

**数据类型**：
```
用户数据：账号、偏好设置、学习档案
内容数据：单词库、例句、语法规则
行为数据：学习时长、错题记录、交互日志
时间序列：每日学习进度、能力变化曲线
```

**信息流入**：
- 用户注册/登录信息
- 学习行为记录
- 作文和对话内容
- 系统日志数据

**信息流出**：
- 用户查询请求结果
- 学习进度统计
- 推荐系统所需数据
- 分析报告数据

---

### 2. 检索引擎 (Meilisearch)

**功能职责**：
- 提供毫秒级词汇模糊搜索
- 支持拼写纠错和自动补全
- 多语言词典快速检索
- 相关性排序和过滤

**核心能力**：
```
容错搜索：typo-tolerance，支持拼写错误
实时索引：数据变更后即时可搜
多语言：中文分词、英文词干提取
过滤 faceting：按词性、难度、主题筛选
```

**信息流入**：
- 从PostgreSQL同步的词汇数据
- 用户搜索查询

**信息流出**：
- 搜索结果列表
- 搜索建议（自动补全）
- 相关词汇推荐

---

### 3. LLM服务 (vLLM + Qwen/DeepSeek)

**功能职责**：
- **词库生成**：根据主题自动生成词汇列表和例句
- **对话核心**：场景化对话的回复生成
- **作文评价**：综合评估作文质量、给出改进建议
- **用户画像总结**：分析学习数据，生成能力评估报告
- **课堂互动**：理解学生问题，生成教学建议

**处理流程**：
```
用户输入 → Prompt工程 → vLLM推理 → 结果后处理 → 返回给用户
                ↓
         上下文管理（多轮对话记忆）
```

**信息流入**：
- 用户对话消息
- 作文文本内容
- 学习行为数据（用于画像）
- 课件文本内容

**信息流出**：
- AI回复文本
- 作文评分和建议
- 用户能力分析报告
- 知识点解释

---

### 4. ASR服务 (faster-whisper)

**功能职责**：
- 实时语音识别（流式处理）
- 支持多语言语音输入
- 语音指令识别（控制命令）
- 对话语音转文本

**技术特点**：
```
流式识别：支持WebRTC实时音频流
多语言：自动检测语言类型
低延迟：端到端延迟 < 300ms
高准确：基于Whisper large-v3
```

**信息流入**：
- 用户语音音频流（WebRTC/WebSocket）
- 音频文件（录音回放）

**信息流出**：
- 识别文本结果
- 置信度分数
- 时间戳对齐信息

---

### 5. TTS服务 (Piper / XTTS)

**功能职责**：
- 文本转自然语音输出
- 支持多语言语音合成
- 语音克隆（个性化AI声音）
- 实时语音播报

**技术特点**：
```
实时性：10倍实时合成速度
自然度：接近真人发音
多音色：支持不同性别、年龄音色
情感控制：可调节语速、语调
```

**信息流入**：
- LLM生成的回复文本
- 系统提示语音
- 课件内容文本

**信息流出**：
- 音频数据流（PCM/MP3）
- 音频文件URL

---

### 6. VLM服务 (LLaVA / Qwen-VL)

**功能职责**：
- 课件图片内容识别和理解
- 提取教学知识点
- 分析图表、公式、示意图
- 辅助生成视觉相关的教学建议

**应用场景**：
```
课件分析：识别PPT/白板内容
图表理解：解释统计图表、流程图
公式识别：数学/科学公式解析
场景感知：识别教学环境状态
```

**信息流入**：
- 课件图片（Base64或URL）
- 截图/拍照图像
- 用户上传的图片

**信息流出**：
- 图片内容描述
- 提取的知识点
- 视觉问答结果

---

## 二、基础设施服务模块（9个）

### 7. 缓存服务 (Redis Cluster)

**功能职责**：
- **热数据缓存**：高频查询词汇、热门内容
- **会话状态**：用户登录状态、WebSocket连接映射
- **分布式锁**：防止并发冲突（如同时修改学习记录）
- **速率限制**：API调用频次控制

**缓存策略**：
```
L1缓存：应用内存（Caffeine）- 最热数据
L2缓存：Redis - 会话、查询结果
L3缓存：CDN - 静态资源
```

**信息流入**：
- 数据库查询结果
- 用户会话信息
- 计算结果（如推荐列表）

**信息流出**：
- 缓存命中数据（避免查库）
- 会话验证结果
- 分布式锁状态

---

### 8. 消息队列 (Apache Kafka / NATS)

**功能职责**：
- **异步任务处理**：作文批改、用户行为分析（耗时操作异步化）
- **削峰填谷**：课堂高峰期流量缓冲
- **服务解耦**：各服务间通过消息通信，降低依赖
- **事件驱动**：学习事件触发后续处理

**消息类型**：
```
学习事件：用户完成练习、提交作文
系统事件：模型更新、配置变更
通知事件：学习提醒、成就达成
```

**信息流入**：
- 业务服务发布的事件消息
- 定时任务触发的消息

**信息流出**：
- 消费者服务接收处理
- 消息持久化存储

---

### 9. 文件存储服务 (MinIO + CDN)

**功能职责**：
- **语音录音存储**：用户口语练习录音
- **课件文档存储**：PDF、PPT、Word等教学资料
- **图片存储**：用户头像、课件截图、作文图片
- **静态资源**：前端资源、语音合成缓存

**存储结构**：
```
/audio/      - 语音录音文件
/documents/  - 课件文档
/images/     - 图片资源
/cache/      - TTS合成缓存
/exports/    - 导出文件（学习报告等）
```

**信息流入**：
- 用户上传的语音、图片、文档
- 系统生成的TTS音频
- 导出的学习报告

**信息流出**：
- 文件访问URL（通过CDN加速）
- 文件元数据信息

---

### 10. 实时通信服务 (Socket.io + Redis Adapter)

**功能职责**：
- **WebSocket连接管理**：维护客户端长连接
- **消息分发**：课堂内广播、私信
- **多端同步**：同一用户多设备状态同步
- **房间管理**：课堂房间、私聊房间

**通信模式**：
```
广播：教师→全体学生
组播：分组讨论
单播：私聊、系统通知
发布订阅：实时数据推送
```

**信息流入**：
- 客户端WebSocket消息
- 服务端业务事件

**信息流出**：
- 推送到客户端的消息
- 连接状态变更通知

---

### 11. 推荐引擎 (LightFM + FAISS)

**功能职责**：
- **个性化内容推荐**：根据用户画像推荐学习内容
- **学习路径规划**：基于知识图谱的最优学习顺序
- **相似用户发现**：找到学习风格相似的用户
- **冷启动处理**：新用户的默认推荐策略

**推荐类型**：
```
内容推荐：单词、文章、练习题
路径推荐：学习顺序、复习计划
用户推荐：学习伙伴、教师匹配
```

**信息流入**：
- 用户行为数据（点击、学习、收藏）
- 内容特征向量
- 用户画像数据

**信息流出**：
- 推荐内容列表
- 推荐置信度分数
- 推荐理由说明

---

### 12. 知识图谱服务 (Neo4j + NLP抽取)

**功能职责**：
- **词汇关系图谱**：同义词、反义词、词根词缀关系
- **知识点关联**：语法点、话题、难度的关联网络
- **学习前置依赖**：识别学习某知识的前置条件
- **智能检索增强**：基于关系的语义搜索

**图谱实体**：
```
节点：单词、语法点、话题、难度级别
关系：同义、反义、包含、前置、相关
属性：词性、难度、使用频率
```

**信息流入**：
- 词汇数据（从PostgreSQL导入）
- NLP抽取的关系
- 用户学习数据（动态更新关系权重）

**信息流出**：
- 词汇关系查询结果
- 学习路径建议
- 知识关联推荐

---

### 13. 监控日志服务 (Prometheus + Grafana + ELK)

**功能职责**：
- **系统性能监控**：CPU、内存、GPU使用率
- **业务指标监控**：QPS、延迟、错误率
- **分布式追踪**：请求链路追踪（Jaeger）
- **日志聚合分析**：集中收集和分析各服务日志

**监控维度**：
```
基础设施：服务器资源、网络状态
应用性能：API响应时间、吞吐量
业务指标：活跃用户、学习完成率
错误追踪：异常捕获、告警通知
```

**信息流入**：
- 各服务暴露的Metrics
- 应用日志输出
- 追踪数据（Trace）

**信息流出**：
- 监控仪表盘（Grafana）
- 告警通知（邮件、短信、钉钉）
- 日志查询结果（Kibana）

---

### 14. 安全认证服务 (Keycloak + Kong)

**功能职责**：
- **用户身份认证**：JWT/OAuth2登录
- **权限控制**：RBAC角色权限管理
- **API限流**：防止恶意请求和DDoS
- **数据加密**：传输加密（TLS）、存储加密
- **审计日志**：安全事件记录

**安全策略**：
```
认证：JWT Token、Refresh Token机制
授权：角色（学生/教师/管理员）+ 权限（读/写/删）
限流：按用户、按IP、按API分级限流
加密：AES-256存储加密、TLS 1.3传输
```

**信息流入**：
- 用户登录凭证
- API访问请求
- 权限变更操作

**信息流出**：
- JWT Token
- 认证结果
- 审计日志

---

### 15. 模型管理服务 (MLflow + Seldon)

**功能职责**：
- **模型版本管理**：LLM、ASR、TTS模型版本控制
- **A/B测试**：新旧模型效果对比
- **模型热更新**：不停机切换模型
- **模型性能监控**：推理延迟、准确率追踪

**管理流程**：
```
模型开发 → 版本注册 → 测试验证 → 灰度发布 → 全量上线 → 监控反馈
```

**信息流入**：
- 新训练好的模型文件
- 模型性能指标
- 用户反馈数据

**信息流出**：
- 模型服务部署配置
- A/B测试结果
- 模型性能报告

---

## 五、关键数据流转总结

| 业务场景 | 主要数据流 | 涉及模块 |
|---------|-----------|---------|
| **词汇查询** | 用户查询 → 缓存检查 → 搜索引擎 → 知识图谱 → 数据库 → 结果聚合 | Redis, Meilisearch, Neo4j, PostgreSQL |
| **AI对话** | 用户语音 → ASR → LLM → TTS → 语音输出 | ASR, LLM, TTS |
| **课堂交互** | 课件截图 → VLM分析 → 知识点提取 → 建议生成 → 实时推送 | VLM, LLM, Socket.io |
| **作文批改** | 提交作文 → 消息队列 → 异步批改 → 结果存储 → 通知用户 | Kafka, GEC, BERT, LLM |
| **用户画像** | 行为采集 → 特征工程 → 知识追踪 → 画像聚合 → 推荐应用 | DKT, LightFM, Neo4j |
| **背诵复习** | 查询复习计划 → FSRS算法 → 返回单词 → 记录结果 → 更新记忆模型 | Redis, PostgreSQL, FSRS |
