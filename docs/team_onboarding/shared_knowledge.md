## 🔧 统一技术规范

### 代码规范
- **语言**: Python 3.11+
- **框架**: FastAPI + SQLAlchemy 2.0
- **异步**: 所有IO操作必须使用async/await
- **类型注解**: 强制使用Type Hints
- **代码格式**: Black + isort + ruff

### API规范
- **协议**: RESTful + WebSocket(实时场景)
- **认证**: JWT Bearer Token
- **响应格式**: 
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {}
  }
  ```
- **错误码**: 4xx客户端错误，5xx服务端错误

### 数据库规范
- **命名**: 小写下划线，复数形式 (users, learning_logs)
- **主键**: 自增ID或UUID
- **时间字段**: created_at, updated_at必备
- **软删除**: 使用deleted_at而非物理删除
- **索引**: 外键必须建索引，查询字段评估后建索引

```
                    ┌─────────────┐
                    │   核心功能G  │

                    └──────┬──────┘
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
      ┌─────────┐    ┌─────────┐    ┌─────────┐

           │               │               │
           └───────────────┼───────────────┘

              ┌───────────────────────┐
              │    A(数据库+ES)       │
              │    B(缓存+队列+存储)   │

```

**关键依赖说明**:
- G依赖E(模型调用)、C(词汇关系)、D(权限验证)
- E依赖A(上下文存储在Redis)
- C依赖A(词汇数据在PostgreSQL)
- 所有模块都依赖B的基础设施

## 🔐 安全红线

1. **密码安全**
   - 必须使用bcrypt/Argon2哈希
   - 禁止日志记录密码
   - 禁止返回密码字段

2. **认证安全**
   - JWT密钥定期轮换
   - Token过期时间合理设置
   - 敏感操作二次确认

3. **数据安全**
   - 用户数据隔离
   - SQL注入防护
   - XSS输出编码

4. **API安全**
   - 限流防暴力破解
   - CORS白名单
   - 敏感接口审计日志

## 🚀 开发工作流

### Git工作流
```
main (生产分支)
  │
  ├── develop (开发分支)
  │     │
  │     ├── feature/xxx (功能分支)
  │     │
  │     └── hotfix/xxx (紧急修复)
  │
  └── release/v1.x (发布分支)
```

### 提交规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式(不影响功能)
refactor: 重构(非feat非fix)
test: 测试相关
chore: 构建/工具相关
```
