# VC引导：核心学习功能开发

## 你的任务
实现AI外语学习系统的三大核心模块：单词、作文、对话。

## 技术栈
- Python + FastAPI
- PaddleOCR (作文图片识别)
- T5-GECToR (语法纠错)
- faster-whisper (语音转文字)
- XTTS (语音合成)

---

## 模块一：词汇模块

### 功能清单
- [ ] 查词API: 支持模糊搜索、拼写纠错
- [ ] 词汇详情: 释义、例句、发音、标签
- [ ] 词库生成: LLM生成主题词汇 + 自动标签
- [ ] 推荐复习: 基于薄弱点的个性化推荐

### 查词流程
```
用户输入 → Redis缓存? → Elasticsearch → 知识图谱 → PostgreSQL
              ↓命中
           直接返回
```

### 关键约束
⚠️ 查词未命中时，调用Kimi API生成词汇信息，并建立知识图谱关系
⚠️ 返回结果必须包含: 近义词、反义词、同根词、形近/音近词

---

## 模块二：作文模块

### 功能清单
- [ ] OCR文本提取 (PaddleOCR)
- [ ] 语料库检查 (LanguageTool API)
- [ ] GEC语法纠错 (T5-GECToR)
- [ ] LLM综合分析 (Kimi API)
- [ ] 多维度评分算法

### 批改流程
```
作文图片 → OCR → 语料库检查 → GEC纠错 → LLM综合分析 → 评分 → 展示
```

### 评分维度
| 维度 | 权重 | 计算方式 |
|------|------|----------|
| 内容 | 30% | LLM评估 |
| 结构 | 25% | 规则引擎 |
| 语言 | 25% | 词汇丰富度+句式多样性 |
| 语法 | 20% | GEC结果+拼写检查 |

### 关键约束
⚠️ 必须使用语料库和语法检查结果作为LLM评分的参考输入
⚠️ OCR优先，低置信度时提示重拍而非切VLM

---

## 模块三：对话模块

### 功能清单
- [ ] 场景扩写: 用户描述 → Kimi API扩写 → 结构化Prompt
- [ ] 对话执行: Qwen本地模型 + 场景System Prompt
- [ ] ASR集成: faster-whisper语音转文字
- [ ] TTS集成: XTTS语音合成

### 双LLM架构
```
用户场景描述 → Kimi API(扩写) → 生成System Prompt → Qwen(对话执行)
```

### 关键约束
⚠️ 场景扩写必须使用Kimi API的thinking模式
⚠️ 对话执行使用本地Qwen2.5-7B，确保低成本+低延迟
⚠️ 上下文保留最近20轮，超限时自动摘要

---

## 验收标准
- 词汇: 查词响应<100ms，推荐准确率>30%
- 作文: 批改全流程<10秒，评分与人工评分相关性>0.8
- 对话: 首Token延迟<500ms，场景扩写准确率>90%
