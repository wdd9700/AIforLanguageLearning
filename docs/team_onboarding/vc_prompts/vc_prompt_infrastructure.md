# VC引导：基础设施层

## 你的任务
搭建系统的高速缓存、异步消息和静态资源存储基础设施。

## 技术栈
- Redis Cluster (缓存 + 分布式锁 + 限流)
- RabbitMQ / Apache Kafka (消息队列)
- MinIO / 阿里云OSS (对象存储)
- Python + celery + redis-py + boto3

## 必须实现的功能

### 缓存层(Redis)
- [ ] 词汇查询结果缓存 (TTL: 1小时)
- [ ] 用户会话状态缓存 (TTL: 30分钟)
- [ ] 热点数据预加载机制
- [ ] 分布式锁实现 (防止缓存击穿)

### 消息队列
- [ ] 作文批改任务队列 (优先级: 高)
- [ ] 词汇生成任务队列 (优先级: 中)
- [ ] 学习报告生成队列 (优先级: 低)
- [ ] 死信队列处理机制

### 文件存储
- [ ] 用户头像/作文图片上传接口
- [ ] 预签名URL生成 (安全访问)
- [ ] 文件类型校验 (仅jpg/png/pdf)
- [ ] 缩略图自动生成

## 关键约束
⚠️ 缓存策略: 读多写少用Cache-Aside，写多读少用Write-Through
⚠️ 消息队列必须保证作文批改任务不丢失
⚠️ 文件上传大小限制: 单文件<10MB

## 验收标准
- Redis命中率 > 85%
- 消息队列消费延迟 < 5秒
- 文件上传接口支持断点续传
