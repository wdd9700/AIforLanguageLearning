# 成员D：安全认证 - 技术栈要求

## 角色定位
负责构建系统的身份认证、权限管理和数据安全防护。

---

## 技术栈深度要求

| 技术 | 深度要求 | 具体掌握内容 |
|------|----------|--------------|
| **JWT / OAuth2** | ⭐⭐⭐⭐⭐ 精通 | Token结构、签名验证、刷新机制、Scope权限 |
| **密码学基础** | ⭐⭐⭐⭐ 熟练 | 哈希算法(bcrypt/Argon2)、对称/非对称加密、盐值 |

## 必须深入理解的概念

### 1. JWT的Header.Payload.Signature结构及验证流程
- Base64Url编码原理
- HMAC-SHA256签名算法
- 公钥/私钥验证(RS256)
- Token过期和刷新策略

### 2. 为什么密码必须用bcrypt而不是MD5/SHA256
- 彩虹表攻击原理
- 盐值(Salt)的作用
- 工作因子(Work Factor)的调整
- 计算密集型哈希的目的

### 3. CSRF攻击原理及Token防御机制
- 同源策略和跨站请求
- CSRF Token的生成和验证
- SameSite Cookie属性
- 双重提交Cookie模式

## 核心技能检查清单

### JWT/OAuth2
- [ ] 能生成和验证JWT令牌
- [ ] 理解Access Token和Refresh Token的分工
- [ ] 掌握OAuth2的授权码流程
- [ ] 能实现Scope级别的权限控制

### 密码安全
- [ ] 能正确使用bcrypt/Argon2哈希密码
- [ ] 理解密码复杂度策略
- [ ] 掌握密码重置的安全流程
- [ ] 了解凭证填充(Credential Stuffing)攻击

### Web安全
- [ ] 能防止SQL注入(参数化查询)
- [ ] 能防止XSS攻击(输入过滤/输出编码)
- [ ] 能防止CSRF攻击(Token验证)
- [ ] 能防止暴力破解(限流/验证码)

### FastAPI安全
- [ ] 能使用Depends实现依赖注入
- [ ] 能编写自定义权限装饰器
- [ ] 掌握OAuth2PasswordBearer的使用
- [ ] 能实现RBAC权限模型
