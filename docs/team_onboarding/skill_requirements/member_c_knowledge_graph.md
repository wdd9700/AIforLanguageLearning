# 成员C：知识图谱和推荐引擎 - 技术栈要求

## 角色定位
负责构建词汇关系网络和个性化学习推荐引擎。

---

## 技术栈深度要求

| 技术 | 深度要求 | 具体掌握内容 |
|------|----------|--------------|
| **Neo4j** | ⭐⭐⭐⭐⭐ 精通 | Cypher查询、图模型设计、索引优化、APOC算法库 |
| **Python + 图算法** | ⭐⭐⭐⭐ 熟练 | NetworkX、图遍历、最短路径、社区发现 |

## 必须深入理解的概念

### 1. 图数据库与传统关系型数据库的差异
- 图遍历 vs 表连接的性能差异
- 节点、关系、属性的建模方式
- 图算法的应用场景

### 2. 协同过滤的冷启动问题
- 新用户冷启动解决方案
- 新物品冷启动解决方案
- 混合推荐策略

### 3. 向量检索的ANN近似最近邻算法
- 暴力搜索 vs ANN的权衡
- HNSW、IVF等索引结构原理
- 召回率与查询速度的平衡

## 核心技能检查清单

### Neo4j
- [ ] 能设计合理的图数据模型
- [ ] 掌握Cypher的MATCH、WHERE、RETURN语法
- [ ] 能创建节点和关系的索引/约束
- [ ] 理解APOC库的常用过程
- [ ] 能进行图算法分析(中心性、社区发现)

### 推荐算法
- [ ] 理解协同过滤(基于用户/基于物品)
- [ ] 掌握矩阵分解(SVD、NMF)原理
- [ ] 能进行特征工程(用户画像、物品特征)
- [ ] 理解推荐系统的评估指标(准确率、召回率、覆盖率)

### 向量检索
- [ ] 理解Embedding的生成方式
- [ ] 掌握FAISS的索引类型选择
- [ ] 能进行向量相似度计算(余弦、欧氏距离)
- [ ] 理解量化(PQ)和降维(PCA)技术
