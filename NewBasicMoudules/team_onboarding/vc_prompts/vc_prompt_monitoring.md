# VC引导：数据采集与监控体系

## 你的任务
构建全链路用户行为追踪和系统监控告警体系。

## 技术栈
- ClickHouse / TimescaleDB (时序数据)
- Grafana + Prometheus (监控可视化)
- ELK Stack (日志聚合)
- Python + sentry-sdk + prometheus-client

## 必须实现的功能

### 行为数据采集
- [ ] 事件追踪: 查词、开始对话、提交作文、完成测试
- [ ] 性能指标: API响应时间、模型推理耗时
- [ ] 用户画像数据: 学习时长、活跃时段、偏好内容
- [ ] 漏斗分析: 注册→首次学习→持续学习→付费转化

### 系统监控
- [ ] API接口监控 (QPS、错误率、P99延迟)
- [ ] 模型服务监控 (调用次数、Token消耗、失败率)
- [ ] 数据库监控 (连接数、慢查询、磁盘使用)
- [ ] 基础设施监控 (CPU、内存、磁盘、网络)

### 日志体系
- [ ] 结构化日志 (JSON格式)
- [ ] 日志分级: DEBUG/INFO/WARNING/ERROR/CRITICAL
- [ ] 分布式追踪 (TraceID贯穿请求链路)
- [ ] 错误自动上报 (Sentry集成)

## 关键约束
⚠️ 用户敏感信息(密码、Token)禁止记录到日志
⚠️ 监控数据采集不能影响主业务流程 (异步发送)
⚠️ 日志保留策略: 原始日志7天，聚合数据90天，统计数据永久
⚠️ 告警阈值: API错误率>5%或P99延迟>2s时触发

## 核心监控指标
| 指标 | 目标值 | 告警阈值 |
|------|--------|----------|
| API可用性 | 99.9% | <99% |
| P99延迟 | <500ms | >2s |
| 模型调用成功率 | 99% | <95% |
| 数据库连接池 | <80% | >90% |

## 验收标准
- 全链路追踪可追溯任意请求
- 告警响应时间 < 5分钟
- 监控Dashboard实时刷新 < 10秒
- 日志查询支持秒级检索
