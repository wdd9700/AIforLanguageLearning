## 🐛 已知问题清单

### 问题1: 学习路径数据覆盖度不足

**描述**:  
查询 `happy → excited` 的学习路径时返回空结果

**原因**:  
知识图谱中缺少连接这两个词汇的中间节点（如 `happy → joyful → cheerful → excited`）

**影响**:  
- 严重程度: 低
- 影响范围: 特定词汇对
- 用户体验: 部分路径无法生成

**当前状态**:  
- [x] 问题已识别
- [ ] 数据补充中

**解决方案**:  
1. 后续Sprint继续导入更多词汇关系数据
2. 添加自动关系发现机制
3. 使用LLM辅助生成关系

**预计解决时间**: Sprint 5-6

---

### 问题2: 可选依赖降级处理

**描述**:  
LightFM/FAISS未安装时，推荐系统降级到基于规则的推荐

**原因**:  
降低部署复杂度，避免强制安装重型依赖

**影响**:  
- 严重程度: 低
- 影响范围: 推荐质量（无LightFM时）
- 用户体验: 几乎无感知

**当前状态**:  
- [x] 降级处理已实现
- [ ] 生产环境建议安装

**解决方案**:  
```bash
# 生产环境建议安装
pip install lightfm faiss-cpu
```

**预计解决时间**: 部署阶段

### 问题3: LLM服务依赖

**描述**:

**原因**:  
LLM服务独立运行，可能未启动或超时

**影响**:  
- 严重程度: 低
- 影响范围: 查词功能
- 用户体验: 知识图谱功能正常工作，仅缺少LLM生成内容

**当前状态**:  
- [x] 错误处理已添加
- [ ] LLM服务启动后自动恢复

**解决方案**:  
1. 启动LLM服务
2. 添加LLM服务健康检查
3. 添加缓存机制避免重复请求

**预计解决时间**: 集成测试阶段

## 💬 给组长的说明

### 关于问题1（学习路径）

**这不是Bug，是数据覆盖度问题**。

类比：就像地图软件刚上线时某些小路没有数据，但随着用户数据积累会自动完善。

**当前处理**: 
- 返回 `found: false` 明确告知
- 不影响其他功能
- 后续数据导入后自动解决

### 关于问题2（可选依赖）

**这是设计选择，不是缺陷**。

核心功能（知识图谱查询、基于规则的推荐）**完全不依赖** LightFM/FAISS。

安装这些库只是**增强**推荐效果，系统会自动检测并使用。

### 关于问题3（LLM）

**知识图谱模块独立工作**。

即使LLM不可用，以下功能完全正常：
- ✅ 词汇关系查询
- ✅ 智能推荐
- ✅ 同根词识别
- ✅ 学习路径生成
