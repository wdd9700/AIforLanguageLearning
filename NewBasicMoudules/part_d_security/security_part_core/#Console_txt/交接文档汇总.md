## 模块概述

安全认证模块（Security Part）是AI外语学习系统的身份认证与权限管理组件，基于FastAPI实现，提供完整的JWT认证、RBAC权限控制和安全防护机制。

### 核心功能

- ? JWT双令牌机制（Access Token + Refresh Token）
- ? HttpOnly Cookie存储，防XSS攻击
- ? bcrypt密码哈希（成本因子12）
- ? 速率限制防暴力破解
- ? CSRF双重验证
- ? RBAC三级权限（学生/教师/管理员）
- ? Token撤销机制
- ? 敏感操作二次确认

## 快速接入指南

### 3分钟接入你的模块

```python
from security_part.auth_routes import get_current_user
from security_part.rbac import Role, require_role
from fastapi import APIRouter, Depends

router = APIRouter()

# 基础认证 - 任何登录用户可访问
@router.get("/my-resource")
async def get_resource(current_user: dict = Depends(get_current_user)):
    return {"user": current_user["username"], "data": "..."}

# 角色控制 - 仅教师和管理员
@router.post("/grade-essay")
@require_role(Role.TEACHER)
async def grade_essay(data: dict, current_user: dict = Depends(get_current_user)):
    return {"graded_by": current_user["username"]}

# 管理员专用
@router.delete("/system/config")
@require_role(Role.ADMIN)
async def delete_config(current_user: dict = Depends(get_current_user)):
    return {"deleted_by": current_user["username"]}
```

## API接口文档

### 公开接口（无需认证）

| 方法 | 路径 | 说明 | 请求体 |
|------|------|------|--------|
| POST | `/auth/register` | 用户注册 | `{"username":"","password":"","email":""}` |
| POST | `/auth/login` | 用户登录 | `{"username":"","password":""}` |
| GET | `/health` | 健康检查 | - |

### 需要认证（Cookie自动携带）

| 方法 | 路径 | 说明 | CSRF Token |
|------|------|------|------------|
| GET | `/auth/me` | 获取当前用户信息 | 不需要 |
| POST | `/auth/refresh` | 刷新Access Token | 不需要 |
| POST | `/auth/logout` | 用户登出 | 需要 |
| POST | `/auth/change-password` | 修改密码 | 需要 |
| DELETE | `/auth/account` | 删除账号 | 需要 |

### 管理员专用

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/auth/admin/users` | 列出所有用户 |
| DELETE | `/auth/admin/users/{username}` | 删除指定用户 |

### 响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 错误码

| 状态码 | 含义 | 常见场景 |
|--------|------|----------|
| 400 | 请求参数错误 | 密码复杂度不够 |
| 401 | 未认证 | Token过期/无效 |
| 403 | 无权限 | 普通用户访问管理员接口 |
| 429 | 请求过多 | 登录次数超限 |

## 常见问题

### Q1: 前端如何对接？

```javascript
// 1. 登录（保存Cookie）
const res = await fetch('/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username, password }),
  credentials: 'include'  // 关键：携带Cookie
});
const data = await res.json();
window.csrfToken = data.data.csrf_token;  // 保存CSRF Token

// 2. 发送请求（自动携带Cookie）
await fetch('/api/something', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-CSRF-Token': window.csrfToken,  // POST需要
  },
  credentials: 'include',
  body: JSON.stringify(data)
});
```

### Q2: 401 Unauthorized 怎么办？

排查步骤：
1. 前端是否设置 `credentials: 'include'`?
2. 后端CORS是否配置 `allow_credentials=True`?
3. Token是否过期？调用 `/auth/refresh` 刷新
4. Cookie是否被浏览器阻止？检查SameSite设置

### Q3: Redis连接失败？

```bash
# 快速启动Redis
docker run -d -p 6379:6379 redis:7-alpine

# 系统会自动回退到内存存储（仅单实例可用）
```

### Q4: 如何创建管理员账号？

目前需要直接修改数据库：
```sql
UPDATE users SET role = 'admin' WHERE username = 'your_user';
```

后续会添加管理员创建接口。

## 各模块对接要点

### 成员A - 数据库/ES模块

- 所有查询必须带 `user_id` 过滤，实现数据隔离
- 使用 `get_current_user` 获取当前用户信息

### 成员B - 基础设施模块

- 健康检查 `/health` 无需认证
- 管理接口使用 `@require_role(Role.ADMIN)`

### 成员C - 知识图谱模块

- 使用 `@require_permission("graph:read:basic")` 控制图谱访问
- 不同角色可访问不同深度的图谱

### 成员E - 模型路由模块

- 根据用户角色设置不同的AI调用限流
- 学生: 10 RPM, 教师: 30 RPM, 管理员: 100 RPM

### 成员F - 监控模块

- 记录审计日志：用户操作、敏感操作
- 使用 `logger = logging.getLogger("audit")`

### 成员G - 核心功能模块

- 使用 `router = APIRouter(dependencies=[Depends(get_current_user)])` 统一加认证
- 作文批改等教师功能使用 `@require_role(Role.TEACHER)`
